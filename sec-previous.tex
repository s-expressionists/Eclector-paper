\section{Existing \cl{} readers}

\subsection{Embeddable Common Lisp (ECL)}

ECL is implemented as a core written in \clanguage{}.  It is built by
first compiling this core and then adding code written in \cl{} by
loading it into the system resulting from the compilation of the core.
For that reason, the reader must be written largely in \clanguage{}. 

When an attempt is made to read an incomplete expression, so that the
\texttt{end-of-file} condition is signaled, no restart other than
\texttt{restart-toplevel} is available.

Similarly when an attempt is made to read a symbol with an explicit
package marker where the corresponding package does not exist, an
error of type \texttt{simple-error} is signaled, and again, the only
restart available is \texttt{restart-toplevel}.

When an attempt is made to read an incorrect token in the form of a
malformed symbol with three package markers, again an error of type
\texttt{simple-error} is signaled, and the message is cryptic.  For
example, an attempt to read the malformed symbol \texttt{cl:::car},
the error message is ``There is no package with the name CL:::C''.

One might think that the reason ECL proposes so few restarts is
because it might difficult to do so for a function written in
\clanguage{} but that is not the case; ECL has full support for
functionality analogous to \texttt{restart-case} for \clanguage{}
code.

\subsection{Steel-Bank Common Lisp (SBCL)}

The SBCL reader is written in \cl{}, without the use of the specific
features of the \cl{} Object System (\clos{}) such as generic
functions and standard classes.

When an attempt is made to read an incomplete expression, so that the
\texttt{end-of-file} condition is signaled, no restart other than
\texttt{retry} \texttt{abort} is available.

When an attempt is made to read a symbol with an explicit package
marker where the corresponding package does not exist, an error of
type \texttt{simple-reader-package-error} is signaled.  Several
restarts are then available:

\begin{itemize}
\item using the current package instead,
\item retry finding the package (presumably after it has been created
  by the user),
\item specify a different package,
\item return an uninterned symbol, and
\item specify a symbol to return.
\end{itemize}

When an attempt is made to read an incorrect token in the form of a
malformed symbol with three package markers, an error of type
\texttt{simple-reader-error} is signaled, and no restart other than
\texttt{retry} \texttt{abort} is available.

\subsection{Clozure Common Lisp (CCL)}

When an attempt is made to read an incomplete expression, so that the
\texttt{end-of-file} condition is signaled, only restarts that return
to previous break levels are available.

When an attempt is made to read a symbol with an explicit package
marker where the corresponding package does not exist, several
additional restarts are available:

\begin{itemize}
\item supplying a different name for the package
\item retry finding the package (presumably after it has been created
  by the user),
\item making the supplied package name a nickname for the package
  \texttt{common-lisp-user}.
\end{itemize}

\subsection{Clasp}

Clasp is a \cl{} implementation that was specifically designed to
cooperate (FIXME, that's not the right term) with \cplusplus{} code,
in order to take advantage of existing libraries written in
\cplusplus{}.  Clasp was originally written as a conversion of the
\clanguage{} code of ECL to \cplusplus{} and with almost all the \cl{}
code from ECL.  As such, its reader is a \cplusplus{} conversion of
the reader from ECL.  However, in later bootstrapping stages, Clasp
uses Eclector, the subject of this paper, in order to make Eclector
the default reader.

There are plans to modify the bootstrapping procedure of Clasp so that
the initial \cplusplus{} reader is no longer needed.  For all these
reasons, we do not review the \cplusplus{} reader of Clasp, and
Eclector is fully described in \refSec{sec-eclector}.
